# Security Policy

At ZakatFlow, we take the security and privacy of our users' financial and religious data very seriously. We appreciate your help in keeping our platform safe.

## Supported Versions

Currently, we provide security updates for the following versions of ZakatFlow:

| Version | Supported          |
| ------- | ------------------ |
| v0.32.x | :white_check_mark: |
| < 0.32.x| :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you believe you have found a security vulnerability in ZakatFlow, please report it to us privately so we can address it before it can be exploited.

### What to Include

When reporting a vulnerability, please provide as much context as possible to help us reproduce and fix the issue quickly:
- A detailed description of the vulnerability and its potential impact.
- Step-by-step instructions to reproduce the issue.
- Details about your environment (e.g., browser, OS, environment variables).
- Any relevant logs, screenshots, or code snippets.

### What to Expect

1. **Acknowledgment:** We will acknowledge receipt of your vulnerability report within 48 hours.
2. **Investigation:** We will investigate the issue and confirm whether the vulnerability is valid. We may ask for further information or clarification during this phase.
3. **Resolution:** If the vulnerability is confirmed, we will work on a patch and coordinate a release.
4. **Disclosure:** Once the vulnerability is patched and deployed, we may publicly acknowledge your contribution (with your permission) in our release notes or a security advisory.

Thank you for helping to protect the ZakatFlow community!
